  • Online backup
  • Offline backup (TBD)

  • Incremental backup (TBD)

Backup folder layout ([F] means file, [D] means directory)

  • [D] User-specified directory
    • [D] container-backup@job_name

      • [F] image.tar

Targets¶

POST /api/v1/target/¶

Create a backup remote target.

Request JSON Object:
 
  • name (string) – name. [required]
  • ipaddr (string) – ip address. [required]
  • desc (string) – description
  • username (string) – username. [required]
  • password (string) – password. [required]
  • base (string) – base directory of target
  • encrypt (boolean) – encrypt of access target
Response JSON Object:
 
  • object – GET /api/v1/target/(int:id)

Example request

$ curl -sq -XPOST -b cookies.txt -d \
    '{
        "name": "testsite",
        "ipaddr": "127.0.0.1:5000",
        "username": "admin",
        "password": "admin",
        "base": "/"
    }' http://${QIP}:${QPORT}/api/v1/target/

Example response

{
    "base": "/",
    "desc": "",
    "encrypt": false,
    "id": 1,
    "ipaddr": "127.0.0.1:5000",
    "name": "testsite",
    "username": "admin"
}

Backup¶

POST /api/v1/backup/¶

Create a backup schedule

Request JSON Object:
 
  • job_name (string) – backup job name [required]
  • container_id (string) – container ID [required]
  • type (string) – container type [required]
  • dest (object) – [required]
  • target (string) – one of local, remote
  • path (string) – destination path to put backup file.
  • profile (int) – remote target ID
  • at (object) – [required]
  • repeat (string) – one of daily, weekly, monthly, oneshot. [required]
  • disable (boolean) – do not schedule this job
  • retention (int) – from 2 to MAX_INT. 0 means unlimited. [required]
  • features (array) – the array must be in compress, pause.
  • state (object) –
  • code (string) – one of init, scheduled, waiting, running, completed
  • result (int) – 0 means success
  • last_run (int) – last run this job
  • next_run (int) – next run this job
  • progress (int) – running progress [-1, 99]. -1 means no available progress.

daily parameters

Request JSON Object:
 
  • start (int) – time to start a task in seconds starting from 00:00.

weekly parameters

Request JSON Object:
 
  • start (int) – refer to previous definition
  • days (array) – array of week day from 0 (Sunday) to 6 (Saturday).

monthly parameters

Request JSON Object:
 
  • start (int) – refer to previous definition
  • days (array) – array of month day from 1 to 31.

oneshot parameters

Request JSON Object:
 
  • start (int) – epoch

Example request

$ curl -sq -XPOST -b cookies.txt -d \
    '{
        "job_name": "LocalJob",
        "container_id": "utest",
        "type": "lxc",
        "dest": {"target": "local", "path": "/Public/backup"},
        "at": {"repeat": "daily", "start": 60},
        "retention": 3
    }' http://${QIP}:${QPORT}/api/v1/backup/

Example response

{
    "at": {
        "repeat": "daily",
        "start": 60
    },
    "container_id": "utest",
    "dest": {
        "path": "/Public/backup",
        "target": "local"
    },
    "disable": false,
    "features": [],
    "id": 1,
    "job_name": "LocalJob",
    "retention": 3,
    "state": {
        "code": "init",
        "next_run": "2017-03-04 01:00:00"
    },
    "type": "lxc"
}

Restore¶

POST /api/v1/restore/¶

Create a restore job

Request JSON Object:
 
  • job_name (string) – restore job name [required]
  • container_id (string) – container ID [required]
  • type (string) – container type [required]
  • src (object) – [required]
  • target (string) – one of local, remote
  • path (string) – destination path to put backup file.
  • portfolio (string) –
  • disable (boolean) – do not schedule this job
  • state (object) –
  • code (string) – one of init, scheduled, waiting, running, completed
  • result (int) – 0 means success
  • start (int) – time of starting in epoch
  • end (int) – time of completion in epoch
  • progress (int) – running progress [-1, 99]. -1 means no available progress.

Example request

$ curl -sq -XPOST -b cookies.txt -d \
    '{
        "job_name": "RestoreLocalJob",
        "container_id": "utest",
        "type": "lxc",
        "src": {"target": "local", "path": "/Public/backup", "portfolio": "1"}
    }' http://${QIP}:${QPORT}/api/v1/restore/

Example response

{
    "container_id": "utest",
    "disable": false,
    "id": 1,
    "src": {
        "path": "/Public/backup",
        "portfolio": "1",
        "target": "local"
    },
    "state": {
        "code": "init"
    },
    "type": "lxc"
}
